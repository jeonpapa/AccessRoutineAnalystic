# 예측 룰 자가 학습 시스템 — Prediction Rules

> **목적**: 매 차수 D+1 시점에 사전 예측과 실제 결과를 자동 비교(audit)해서 예측 룰의 weight를 보정하거나 신규 룰을 등록한다. 차수가 누적될수록 예측 정확도가 자동으로 향상.

## 자가 학습 사이클

```
D-2 (사전 예측)
  → 각 예측 약물의 근거 룰(rule_id) + 신호 source 명시
     ↓
D+0 (HIRA 공식 결과 공개)
  → 실제 안건 + 통과/미통과 list 확정
     ↓
D+1 (자동 audit)
  → 예측 vs 실제 비교 → TP/FP/FN 분류
  → 각 룰의 hit/miss 기록 누적
     ↓
룰 보정/신규 등록
  → FP 발생 룰 weight 감점 (-0.05)
  → 누적 FP ≥ 3 → 룰 retire 또는 조건 강화
  → FN 발생 → 신규 패턴 분석 → 후보 룰 등록 → 2회 hit 검증 → 정식 등록
```

## 룰 구조 (YAML 형식)

각 룰은 `예측_룰/active/PR-XXX.md` 파일로 저장.

```yaml

rule_id: PR-001
name: "정부 D-14 사무관 직접 거명 → 통과 강한 상관"

last_calibrated: 2026-05-30

weight: 0.85
condition: "약사공론 또는 메디칼타임즈에 D-7~D-14 시점 정부 사무관·국과장 약물명 직접 거명"
prediction: "ON_AGENDA + 통과 가능성 ↑"
```

## 신규 룰 등록 프로세스

```
D+1 audit에서 FN 발견
  ↓
"왜 안건됐는지" 매체 사유 분석
  ↓
공통 패턴 추출 (신호 source + 시점 + 조건)
  ↓
CANDIDATE 룰로 등록 (예측_룰/candidate/PR-NEW.md)
  ↓
다음 1~2차수에서 검증
  ↓
2회 hit (TP) 시 → ACTIVE 승격
1회 FP 시 → 조건 강화 또는 폐기
```

## 룰 폐기 (Retirement) 프로세스

```
누적 FP ≥ 3 → 일시 정지 (PAUSED)
  ↓
근거 재분석 (signal 변화? 정책 환경 변화?)
  ↓
조건 강화 가능 → RESUME
조건 강화 불가 → RETIRED (예측_룰/retired/로 이동)
```
